About this project

Human voice is the most beautiful instrument of all, but the most difficult to play
Richard Strauss

The scope of this project is to show a simple implementation of a Web Speech API, but using a complex architecture to show some different utilities of web APIs also showing the attendees of this workshop a way to achieve backendless complex functionalities.

Aims for this workshop

  • Learn a bit about Web APIs
  • Learn about backendless architectures
  • Achieve a way to use the Web Speech API trough proxies
  • Involve attendees in accessibility common issues
  • Show real usable projects
  • Show some utilities from OS to solve web common issues under a specific context

Keep in mind

Usage constrains

It's important to know that Web Speech API is not fully supported, this is an important constraint when you want to add this kind of features, still there other options to have speech utilities, such as Web Assembly, but the complexity associated to "WA" is significantly higher, keeping that in mind; web speech API is still a pretty good way to handle with speech needs

Position in critical path

If you need a feature like this, generally your application is "working over wheels", it's never a good idea to start your development critical path having features like what Web Speech API offers. This recommendation comes for experience, but if there's any business characteristic you need to solve using something such as this it might be better the "WA" approach
